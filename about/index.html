About Me

I'm a lifelong student and I've been teaching myself about anything and everything since I was a kid.

My first foray into web development came when I was 8 years old, and I've been programming off and on ever since. However, it was only fairly recently that I became serious about development and began studying computer science (loosely following the OSSU curriculum). In fall 2021, I enrolled in Nucamp's full-stack web development program. After graduating from Nucamp in early 2022, I continued my education via Full Stack Open, as well as through independent study and practice.

Never satisfied with simply memorizing and following procedures, I approach software development with an eye to how things work, from the ground up. I strive always to understand the fundamentals and think problems through from first principles.

More about me

Selected Courses and Books

I worked through the classic SICP in its entirety and it transformed my understanding of abstraction and complexity, and gave me my first real grounding in functional programming.

Some highlights amongst the exercises include building a Scheme interpreter, a register machine simulator, a register machine implementation of the Scheme interpreter, and a Scheme-to-register-machine compiler. I then compiled the original Scheme interpreter, leaving me a Scheme interpreter running on a register machine simulator running on a Scheme interpreter.

Another exercise I enjoyed was implementing Church numerals and an addition operator that works with them. Church numerals are a number type defined solely in terms of functions, with no number primitive involved. This was when it really landed for me that you can build a full programming language with only a single primitive data type (i.e., functions). I also wrote a function that un-Churches a Church numeral back into an Arabic numeral.

I always love to see how everything fits together, so naturally I enjoyed building a simulated computer from logic gates all the way up to a simple Reversi game (i.e., from Nand to Tetris, substituting Reversi). This included building:

  • A computer architecture, written in a hardware description language.
  • An assembler (written in Java) for "Hack" assembly code.
  • A compiler (also written in Java) for the Java-like programming language "Jack", which translates Jack into Hack (via an intermediate stack-based virtual machine code).
  • A Reversi game, written in Jack. This gave me some good memory management practice.

Dan Grossman's Programming Languages MOOC clarified a lot of functional programming concepts for me and helped me to understand both functional and object-oriented programming better through comparison.

One exercise that I found especially illuminating was to build the same program twice, once in an object-oriented style and again in a functional style. The course is taught with Standard ML, Racket, and Ruby, and the concepts apply universally.
